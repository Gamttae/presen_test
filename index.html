Lecture 06-A · 재귀 (Recursion)

재귀 Recursion
정의
함수가 자기 자신을 호출하는 기법.
쉽게
큰 문제를 똑같이 생긴 더 작은 문제로 쪼개서 푼다. (마트료시카 인형, 거울 속 거울)
어떻게
def fact(n):           # n! = n*(n-1)*...*1
    if n == 1:         # ① 멈추는 곳
        return 1
    return n * fact(n-1)  # ② 자기 자신을 더 작게 호출

print(fact(4))         # 24
핵심
모든 재귀 = ① base case + ② recursive step 두 부분. 이 둘만 찾으면 끝.
base case 기저 사례 · 멈추는 곳
정의
더 이상 쪼개지 않고 즉시 답을 주는 종료 조건.
쉽게
계단을 내려가다 “여기서 멈춰!” 하는 맨 아래 칸.
어떻게
if n == 1:      # 이게 base case
    return 1
핵심
base case가 없거나 못 만나면 무한 재귀 → RecursionError. 피보나치처럼 base case가 2개일 수도 있다(n==0 or n==1).
recursive step 재귀 단계
정의
문제를 더 작은 같은 문제로 줄여 자기 자신을 부르는 부분.
쉽게
“나머지는 더 작은 나한테 맡길게” — 입력을 한 단계 작게 해서 다시 호출.
어떻게
return n * fact(n-1)       # n-1 로 더 작게
return s[0]==s[-1] and isPal(s[1:-1])  # 양끝 떼고 가운데로
핵심
반드시 입력이 작아져야 base case에 도달한다(안 작아지면 무한반복).
호출 스코프 각 호출은 독립
정의
재귀 호출은 매번 자기만의 공간(스택 프레임)을 만든다.
쉽게
호출마다 변수 n이 따로따로라 서로 안 섞인다. 값을 돌려주면 직전 호출로 돌아간다.
어떻게
def mystery(n):
    if n == 0: return
    print(n)
    mystery(n-1)
    print(n)
mystery(3)   # 출력:  3 2 1 1 2 3   (내려갔다 올라오며 두 번)
핵심
“출력 순서 추적” 문제 단골. 내려갈 때 한 번, 되돌아올 때 한 번 실행됨을 기억.
반복 vs 재귀 loop vs recursion
정의
같은 일을 while/for로(반복) 또는 자기호출로(재귀) 할 수 있다.
어떻게
# 반복
def mult_iter(a, b):
    result = 0
    while b > 0:
        result += a; b -= 1
    return result
# 재귀
def mult(a, b):
    if b == 1: return a
    return a + mult(a, b-1)
핵심
재귀는 간결·직관적이지만 호출 비용·중복계산으로 느릴 수 있다(예: 피보나치).
중첩 함수(보조 함수) helper function
정의
함수 안에 함수를 정의해서 재귀 부분을 따로 둔 형태.
쉽게
바깥은 “준비”, 안쪽은 “진짜 재귀”. 회문(palindrome) 예제가 대표.
어떻게
def isPalindrome(s):
    def toChars(s):              # 준비: 소문자 + 알파벳만
        s = s.lower(); ans = ''
        for c in s:
            if c in 'abcdefghijklmnopqrstuvwxyz': ans += c
        return ans
    def isPal(s):                # 진짜 재귀
        if len(s) <= 1: return True
        return s[0] == s[-1] and isPal(s[1:-1])
    return isPal(toChars(s))
핵심
하노이 탑도 재귀 대표 예: “위 n-1개 옮기고 → 큰 1개 옮기고 → n-1개 다시”.

Lecture 06-B · 딕셔너리 (Dictionary)

딕셔너리 dict · {key: value}
정의
key(열쇠)–value(값) 쌍을 모아두는 자료형.
쉽게
국어사전처럼 “단어(key)로 뜻(value)을 바로 찾는다”. 리스트는 번호(0,1,2)로 찾지만 딕셔너리는 아무 이름표로 찾음.
어떻게
grades = {'Ana':'B', 'John':'A+'}   # 만들기
grades['John']            # 조회 → 'A+'
empty = {}                # 빈 딕셔너리
핵심
순서 개념이 없다(번호로 못 찾음). 이름표(key)로만 찾는다.
key 규칙 / value 규칙
정의
key는 불변(immutable)·유일해야 하고, value는 아무거나 가능.
어떻게
가능불가
keyint, float, str, tuple, bool (유일)list (가변이라 ✕)
value무엇이든 (중복 OK, 리스트·딕셔너리도)제한 없음
핵심
시험 단골: “key로 list를 쓸 수 있나?” → 안 됨(가변이라 hashable 아님).
기본 연산 (추가·조회·삭제·검사)
어떻게
d = {'Ana':'B', 'John':'A+'}
d['Sylvan'] = 'A'     # 추가 / 이미 있으면 수정
d['John']             # 조회 → 'A+'   (없는 key면 KeyError!)
'John' in d           # 있는지 검사 → True
del(d['Ana'])         # 삭제
d.keys()              # 모든 key   (순회 가능)
d.values()            # 모든 value
핵심
없는 key를 d['x']로 꺼내면 KeyError. 그래서 먼저 'x' in d로 확인.
빈도 세기 패턴 counting
정의
각 항목이 몇 번 나왔는지 딕셔너리로 센다.
어떻게
def count(items):
    d = {}
    for x in items:
        if x in d:        # 이미 있으면
            d[x] += 1
        else:             # 처음 보면
            d[x] = 1
    return d
# 한 줄 버전:  d[x] = d.get(x, 0) + 1
핵심
가장 많이 쓰는 패턴(가사 단어 세기 예제). d.get(x,0)+1 한 줄도 꼭 외우기.
메모이제이션 memoization
정의
한 번 계산한 값을 딕셔너리에 저장해 두고, 다음엔 꺼내 쓰는 것.
쉽게
“이미 푼 문제는 다시 안 푼다” → 재귀를 엄청 빠르게.
어떻게
def fib(n, d):
    if n in d:            # 저장돼 있으면 바로 반환
        return d[n]
    ans = fib(n-1, d) + fib(n-2, d)
    d[n] = ans            # 계산 결과 저장
    return ans

print(fib(6, {1:1, 2:2}))   # 13
핵심
fib(34): 1140만 번 → 65번으로 감소. 단 입력이 같으면 항상 같은 결과인 함수에만 안전.

Lecture 07-A · 테스팅 & 디버깅

테스팅 vs 디버깅 testing vs debugging
정의
테스팅=입력↔출력을 스펙과 비교해 “문제가 있나?” 찾기. 디버깅=“왜 틀렸나? 어떻게 고치나?” 원인 추적.
쉽게
테스팅 = 검사(불량 찾기), 디버깅 = 수리(원인 고치기).
핵심
방어적 프로그래밍 = 미리 대비(명세 작성·모듈화·조건 검사). “수프에 벌레 안 빠지게 뚜껑 닫기”.
테스트 종류 unit / regression / integration
어떻게
종류뜻 (쉽게)
Unit 단위함수 하나씩 따로 검사
Regression 회귀고친 버그가 다시 안 생기는지 테스트 추가
Integration 통합전체가 잘 맞물려 도는지
핵심
이름과 뜻 매칭 문제 단골. “함수 하나=unit, 전체=integration”.
Black box vs Glass box 테스트
정의
black box=코드 안 보고 설명서(명세)만 보고 테스트. glass box=코드 속을 보고 모든 경로를 테스트.
쉽게
black=상자 밖에서, glass=투명상자 안을 들여다보며.
핵심
black box는 경계값(빈 리스트·0·아주 큰 수)을 꼭 넣어 본다. glass box가 모든 경로를 거치면 path-complete.
디버깅 방법 print + 이분 탐색
정의
print로 중간 값을 찍어 버그 위치를 좁히는 것.
쉽게
코드 한가운데에 print를 찍어 보고, 버그가 앞쪽인지 뒤쪽인지 반씩 좁혀간다(이분 탐색).
어떻게
def f(x):
    print("진입:", x)     # 함수 들어올 때
    y = x * 2
    print("중간:", y)     # 중간 값
    return y
핵심
“왜 되지?”가 아니라 “왜 이 결과가 나왔지?”를 가설→실험으로. 함수 단위로 작성·테스트·디버그.
에러(예외) 종류 error types
정의
실행 중 문제가 생기면 파이썬이 종류별 에러를 낸다.
어떻게
코드 예에러뜻
[1,2,3][5]IndexError리스트 범위 초과
d['없는키']KeyError딕셔너리에 없는 key

'3'/4TypeError타입이 안 맞음

괄호 안 닫음SyntaxError문법 자체가 틀림
핵심
“이 코드는 무슨 에러?” 매칭 문제 빈출. 위 표만 외우면 됨.

Lecture 07-B · 예외 처리 & assert

예외 처리 try / except
정의
에러가 나도 프로그램을 멈추지 않고 대신 처리하는 구조.
쉽게
try 안을 실행하다 에러가 나면 → except로 점프해서 처리하고 → 계속 진행.
어떻게
try:
    a = int(input())
    print(1 / a)
except ValueError:        # 숫자가 아닐 때
    print("숫자를 입력하세요")
except ZeroDivisionError: # 0일 때
    print("0으로 못 나눠요")
핵심
교수님 “한국말로 중요” 강조 부분. 특정 에러별로 except ErrorName: 나눠 처리 가능.
else / finally
정의
else=예외가 안 났을 때만 실행. finally=무슨 일이 있어도 항상 실행.
어떻게
try:
    위험한_일()
except SomeError:
    print("에러 처리")
else:
    print("에러 없을 때만")
finally:
    print("끝나면 항상 (파일 닫기 등 정리)")
핵심
finally는 return·break·에러와 상관없이 무조건 실행. “정리(청소)용”.
raise 직접 예외 일으키기
정의
내가 직접 “이건 에러야!”라고 예외를 발생시키는 것.
어떻게
def f(x):
    if x < 0:
        raise ValueError("음수는 안 돼요")
    return x ** 0.5
핵심
잘못된 값을 슬쩍 넘기지 말고 raise로 분명히 신호하는 게 좋은 방식.
assert 단언·전제조건 검사
정의
“이건 반드시 참이어야 해”라고 확인. 거짓이면 AssertionError로 즉시 멈춤.
쉽게
함수 입구에서 “이상한 입력이면 바로 멈춰!” 하는 검문소.
어떻게
def avg(grades):
    assert len(grades) != 0, '빈 리스트 안 됨'   # 거짓이면 즉시 중단
    return sum(grades) / len(grades)
핵심
형식: assert 조건, "메시지". 주로 입력 검사. 버그를 생긴 그 자리에서 잡는 방어 도구.

Lecture 08 · 객체지향 프로그래밍 (OOP) ★ 제일 중요

객체 object
정의
데이터(상태)와 그걸 다루는 기능(메서드)을 한 덩어리로 묶은 것. 파이썬의 모든 값은 객체.
쉽게
1234는 int 객체, "hi"는 str 객체. 각 객체는 ①타입 ②내부 데이터 ③쓸 수 있는 기능(메서드)을 가짐.
핵심
“객체는 타입(=클래스)의 인스턴스” — 1234는 int의 인스턴스.
클래스 vs 인스턴스 class vs instance
정의
클래스=설계도/틀. 인스턴스=그 틀로 찍어낸 실제 객체.
쉽게
붕어빵 틀(클래스) → 붕어빵들(인스턴스). 강아지란 ‘이렇게 생겼다’ 정의=클래스, 실제 강아지 10마리=인스턴스.
어떻게
class Dog:          # 틀(정의)
    ...
a = Dog()           # 인스턴스 1 (찍어냄)
b = Dog()           # 인스턴스 2
핵심
교수님 “이거 모르면 기말 못 본다” 1순위. 정의(클래스)와 사용(인스턴스 생성)을 구분.
데이터 속성 vs 메서드
정의
데이터 속성=객체가 무엇인지(상태·명사). 메서드=객체가 무엇을 하는지(행동·동사, 그 클래스 전용 함수).
쉽게
좌표라면 → 데이터: x, y / 메서드: ‘두 점 사이 거리 구하기’.
핵심
속성=명사, 메서드=동사로 기억.
클래스 정의 & __init__ & self
정의
class로 틀을 만들고, __init__은 인스턴스를 만들 때 자동 실행되는 초기화 함수. self는 지금 만들어지는 그 객체 자신.
어떻게
class Coordinate(object):
    def __init__(self, x, y):   # 생성자 (자동 호출)
        self.x = x              # self = 이 객체 자신
        self.y = y              # self.x, self.y = 인스턴스 변수
    def distance(self, other):  # 메서드 (첫 인자 항상 self)
        return ((self.x-other.x)**2 + (self.y-other.y)**2)**0.5

c = Coordinate(3, 4)            # __init__ 자동 실행 (self는 자동 전달)
print(c.x)                      # 3   ← 점(.)으로 속성 접근
핵심
① self는 정의할 때 항상 첫 인자, 호출할 땐 안 넘김(자동). ② __init__은 return 없음. ③ self.x = x (그냥 x=x 아님!).
메서드 호출 — 두 방식은 같다
어떻게
c.distance(zero)               # 보통 방식 (self ← c)
Coordinate.distance(c, zero)   # 완전히 동일 (self에 c 직접 전달)
핵심
교수님 강조 포인트. c.m(z) = Class.m(c, z).
__str__ 출력 모양 정하기
정의
print(객체) 할 때 보여줄 문자열을 직접 정하는 특수 메서드.
쉽게
안 정하면 <__main__.Coordinate object at 0x...> 같은 이상한 주소가 나옴.
어떻게
    def __str__(self):
        return "<" + str(self.x) + "," + str(self.y) + ">"
# print(c)  →  <3,4>
핵심
기본 동작을 내 맘대로 바꾸는 것 = 오버라이드(override).
type() / isinstance()
정의
type(x)=x의 타입(클래스) 확인. isinstance(x, C)=x가 C의 인스턴스인지 True/False.
어떻게
print(type(c))                  # 
print(isinstance(c, Coordinate))# True
연산자 오버로딩 특수 메서드
정의
+ - == < 같은 연산자를 내 클래스에 맞게 동작시키는 특수 메서드(이중 밑줄).
어떻게
메서드연산메서드연산
__add__+__eq__==
__sub__-__lt__<
__mul__*__len__len()
class Fraction(object):
    def __init__(self, n, d):
        self.n = n; self.d = d
    def __add__(self, other):       # 분수 덧셈
        return Fraction(self.n*other.d + other.n*self.d, self.d*other.d)
    def __str__(self):
        return str(self.n) + "/" + str(self.d)

print(Fraction(1,2) + Fraction(1,4))   # 6/8  ← __add__ 호출
핵심
보통 같은 타입의 새 객체를 만들어 반환한다(예: Fraction(...)).

Lecture 09 · 상속 (Inheritance) ★ “별표 별표”

getter / setter
정의
데이터 속성을 읽는 함수(getter)와 바꾸는 함수(setter).
쉽게
밖에서 a.age로 직접 만지지 말고 a.get_age()/a.set_age(5)로 다루자.
어떻게
class Animal(object):
    def __init__(self, age):
        self.age = age
        self.name = None
    def get_age(self):           # getter
        return self.age
    def set_age(self, newage):   # setter
        self.age = newage
핵심
왜? 내부 변수명을 바꿔도(age→years) 바깥 코드가 안 깨짐 → 정보 은닉(유지보수 ↑).
정보 은닉 information hiding
정의
내부 구현을 숨기고 정해진 통로(메서드)로만 접근하게 하는 원칙.
핵심
파이썬은 정보 은닉이 약하다: 밖에서 a.age=999로 바꾸거나 a.size="tiny"로 새 속성도 만들 수 있음(가능하지만 나쁜 스타일).
기본 인자 default argument
정의
인자를 안 주면 미리 정해둔 기본값을 사용.
어떻게
def set_name(self, newname=""):   # 기본값 ""
    self.name = newname
a.set_name()          # 이름 = ""
a.set_name("초코")    # 이름 = "초코"
상속 inheritance · 부모→자식
정의
부모 클래스(superclass)의 속성·메서드를 자식 클래스(subclass)가 그대로 물려받는 것.
쉽게
자식은 부모 걸 다 가진 채 ① 새 기능 추가 ② 기존 기능 바꾸기(오버라이드) 가능. (예: Animal → Cat, Person → Student)
어떻게
class Cat(Animal):          # 괄호 안 Animal = 부모
    def speak(self):        # 새 기능 추가
        print("meow")
    # __init__ 안 써도 Animal 것을 그대로 씀
핵심
class 자식(부모): 형태. 같은 일을 두 번 안 쓰려고 상속을 쓴다.
오버라이드 & 메서드 탐색 순서 ★
정의
오버라이드=자식이 부모와 같은 이름의 메서드를 새로 정의해 덮어쓰기.
쉽게
메서드를 부르면 → 지금 클래스부터 찾고 → 없으면 부모, 조부모로 올라가며 → 처음 만난 것을 쓴다.
어떻게
class Animal(object):
    def speak(self): print("...")
class Cat(Animal):
    def speak(self): print("meow")   # 오버라이드
Cat(3).speak()   # meow  (Animal 것 대신 Cat 것)
핵심
교수님 “별표 별표” 강조. 자식 메서드가 부모 것을 가린다.
부모 생성자 호출
정의
자식 __init__ 안에서 부모의 __init__을 불러 부모 부분을 초기화.
어떻게
class Person(Animal):
    def __init__(self, name, age):
        Animal.__init__(self, age)   # ★ 부모 생성자, self 직접 전달
        self.set_name(name)
        self.friends = []
핵심
형태: 부모클래스.__init__(self, ...). self를 꼭 직접 넘긴다.
클래스 변수 vs 인스턴스 변수 ★
정의
클래스 변수=모든 인스턴스가 공유하는 하나의 값(self 없이 선언). 인스턴스 변수=객체마다 따로(self.).
쉽게
주민등록번호 발급기처럼, 다 같이 쓰는 카운터로 객체마다 고유 번호를 매긴다.
어떻게
class Rabbit(Animal):
    tag = 1                      # ★ 클래스 변수 (self 없음, 공유)
    def __init__(self, age):
        Animal.__init__(self, age)
        self.rid = Rabbit.tag    # 인스턴스 변수 (각자)
        Rabbit.tag += 1          # 다음 토끼 위해 +1

a = Rabbit(2); b = Rabbit(3)
print(a.rid, b.rid, Rabbit.tag)  # 1 2 3
핵심
self 없으면 클래스 변수(공유), self. 있으면 인스턴스 변수(독립). 접근은 Rabbit.tag vs self.rid.
import & random
정의
이미 만들어진 모듈(코드 묶음)을 가져다 쓰는 것.
어떻게
import random
r = random.random()    # 0 이상 1 미만 균등 난수
if r < 0.25:
    print("homework")
핵심
import는 파일 맨 위에. random.random() = 0~1 사이 실수.
